What this proves

One FDW implementation, two SQLite runtimes.

  • The native CLI continues to use modernc.org/sqlite.
  • This worker uses go-sqlite3 and the ncruces FDW adapter.
  • Both execute the same Source, Cursor, and planner code.
  • Go-git clones the selected remote into memory using smart HTTP.
  • Depth and branch controls bound browser memory use for large repositories.
